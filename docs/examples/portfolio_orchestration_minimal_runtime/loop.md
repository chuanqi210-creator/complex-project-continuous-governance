# Loop Example: Portfolio Orchestration

## Loop Purpose

- beat_objective: Choose the first module to move after the operating organization exists.
- target_function: Produce a usable, source-backed, reviewable deliverable.
- module_this_beat_serves: model chain or data source chain.
- standing_lane_this_beat_serves: controller lane.
- loop_type: strategy.
- why_not_local_greedy: The beat compares modules and branch parking before choosing a local edit.

## Result

- Forward artifact type: topology_delta.
- Forward artifact path or index update: portfolio map and branch parking ledger.
- Guard result: local metadata branch no longer blocks the whole project.

## Route

- selected_route: CONTINUE.
- next_beat_objective: Run a vertical slice in the highest-leverage active module.
