# Complex Current State

current_version: 2026-07-22

## Recovery Anchor

Complex uses one seven-behavior spine supported by four coupled engineering layers: Prompt Contract, Context Working Set, Runtime Harness, and Progress Loop. The layers diagnose failure; they are not four linear gates.

The active architecture has three universal mechanisms, four supporting practices, and four conditional extensions. Normative role and evidence status remain separate: the core and supporting practices are `tested`; the four conditional extensions remain `screened`; none is `validated`.

## Active Decisions

- Restore one authoritative project state before routing.
- At project start or strategic reframe, use Framework Grilling only for unresolved user-owned forks that can change the destination; inspect facts first and continue with `no_grill_needed` when no qualifying question remains.
- Keep durable Goal and completion criteria stable; keep changing facts in the current working set.
- Treat project nature as a durable prior and let the active phase, module, or work item control the current beat when its work shape differs.
- Define the artifact contract before assigning tools, agents, lanes, threads, or workflows.
- Compile the smallest sufficient topology from manager work, deterministic Harness, temporary workers, recurring lanes, clean evaluation, and exact responsibility handoff.
- Keep one controller responsible for accepted state, integration, routing, and final acceptance when several resources are active.
- When several durable boundaries keep local state, preserve their local authority and let the controller refresh one compact global control projection from versioned local state capsules on meaningful events or measured staleness.
- Move stable repeated judgment toward templates, checkers, tool contracts, or state machines; route assumption-driven mechanical failure back to discovery or strategic judgment.
- Treat the target project's completion predicate, artifact path, schema, fields, values, quantities, and verifier as the executable interface.
- Diagnose Prompt, Context, Harness, Loop, or model failure before adding prompt text.
- Keep project-internal decisions autonomous and place user intervention only at a responsibility-bearing operation or undelegated value judgment.
- Continue non-terminal `next_route` work until the outcome predicate or a true responsibility/tool boundary.
- Keep operating organization, portfolio control, trace appraisal, and external transfer conditional.
- Use evidence verbs precisely: examples illustrate, markers screen, fixtures reproduce, locked runs compare, and repeated real outcomes validate.

## Active Behavior Spine

1. Restore true state and assemble the Context Working Set.
2. Classify project nature as a prior, then route the active work unit.
3. Assign responsibility and decision rights.
4. Define the artifact contract and compile the minimum control, Harness, and topology.
5. Run a target-function Progress Loop that changes an outcome, exposes a decision, or falsifies a route.
6. Deliver for the intended reader and evaluate the outcome.
7. Leave accepted artifacts, layer diagnosis, recovery pointers, and `next_route`.
